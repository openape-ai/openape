## How to Add a New Persona

1. Add a new persona entry to `PERSONAS` array in `persona-catalog.ts`
2. Ensure all required fields are populated:
   - `key`: Unique kebab-case identifier
   - `title`: Human-readable title shown in UI
   - `role`: Structural chart role (`ceo`, `teamlead`, `specialist`, `sanierer`, `other`)
   - `category`: One of the defined categories
   - `icon`: Lucide icon class
   - `summary`: One-line description
   - `coding`: Whether the persona works with code (Forgejo)
   - `recipeRef`: Pinned recipe reference
   - `recipeParams`: Parameters for recipe instantiation
3. Run the build script to regenerate derived files
4. Update this documentation if the auto-generation doesn't cover it

## Assembling a Company via ORG

To assemble a virtual company using these personas:

1. **Navigate to ORG**: Visit [org.openape.ai](https://org.openape.ai)
2. **Select or Create an Organization**: Choose an existing org or create a new one
3. **Spawn Personas**: 
   - Click "Add Member" or "Spawn Persona"
   - Select the desired persona from the catalog (e.g., CEO, Backend Engineer)
   - The system will instantiate the agent with its pinned recipe (`@v0.2.0`)
4. **Configure the Team**: 
   - Assign personas to roles in your org chart
   - Each agent receives its own DDISA identity
   - Agents automatically poll their assigned tasks from [tasks.openape.ai](https://tasks.openape.ai)
5. **Autonomous Operation**: 
   - Agents work independently according to their operating protocol
   - They pick up tasks, execute work, and report back autonomously
   - No per-task prompting required—assign work in the UI and let agents handle it
