# Pause Agents & Nests

Pause agents or the entire nest to stop LLM turns temporarily while keeping agents connected. Paused agents consume zero tokens and can be resumed instantly without re-authentication.

## Use Cases

- **Overnight pause**: Stop agents from running turns during off-hours without disconnecting them from troop chat
- **Temporary suspension**: Pause a specific agent that is misbehaving while keeping others running
- **Maintenance window**: Pause the entire nest during system maintenance

## How It Works

When an agent is paused:
- The agent stays enrolled in the nest
- The WebSocket connection remains active
- No LLM turns are executed (zero tokens consumed)
- The agent can still receive messages in troop chat
- Resume is instant — no respawn, no re-authentication required

The pause state is checked live at each turn dispatch, so changes take effect immediately without restarting the nest.

## Pause an Individual Agent

To pause a specific agent, use the nest registry API:

```javascript
import { setAgentPaused } from '@openape/openape-nest'

// Pause an agent
setAgentPaused('agent-name', true)

// Resume an agent
setAgentPaused('agent-name', false)
```

The pause state is persisted to `~/.openape/nest/agents.json` (or `/var/openape/nest/agents.json` in production).

## Pause the Entire Nest

To pause all agents in the nest at once:

```javascript
import { setNestPaused } from '@openape/openape-nest'

// Pause all agents
setNestPaused(true)

// Resume all agents
setNestPaused(false)
```

The nest-wide pause state is persisted to `~/.openape/nest/nest-state.json`.

## Pause Precedence

The pause check evaluates in this order:
1. If the nest is paused, all agents are paused (regardless of individual state)
2. Otherwise, check the individual agent's pause state

An agent is paused if **either** the nest is paused **or** the agent is individually paused.

## Behavior Details

### What Happens When Paused

- **LLM turns**: Dropped before execution — no tokens consumed
- **WebSocket**: Remains connected — agent stays in troop chat
- **Reconciliation**: Continues running — agents stay enrolled
- **Message handling**: Messages are received but turns are not dispatched

### What Happens When Resumed

- Turns resume immediately — no restart required
- No re-authentication needed — existing tokens remain valid
- Agent picks up where it left off — no state loss

## Implementation Details

The pause feature is enforced at two levels:

1. **Per-agent pause**: Stored in the `AgentEntry` record (`paused` and `pausedAt` fields)
2. **Nest-wide pause**: Stored in `nest-state.json`

The `isAgentPaused(name)` function checks both states and returns `true` if either is active.

### Guard Points

The pause check is applied at:
- `dispatchTurn`: Before enqueuing a message to the thread session
- `tickAll`: Before processing agent turns during reconciliation

Both the openclaw runtime and the default bridge runtime respect the pause flag.

## CLI Usage

The `ape-troop` CLI provides commands to pause and resume agents and nests:

### Pause/Resume an Agent

```bash
ape-troop agents pause <agent_name>
ape-troop agents resume <agent_name>
```

### Pause/Resume a Nest

```bash
ape-troop nests pause --host-id <nest_host_id>
ape-troop nests resume --host-id <nest_host_id>
```

## API Usage

The troop API exposes endpoints for programmatic control:

### Agent Endpoints

- `POST /api/agents/:name/pause` - Pause a specific agent
- `POST /api/agents/:name/resume` - Resume a specific agent

### Nest Endpoints

- `POST /api/nests/:host_id/pause` - Pause all agents in a nest
- `POST /api/nests/:host_id/resume` - Resume all agents in a nest

These endpoints require the `troop:pause-agent` scope.

## UI Controls (Troop App)

The troop web app provides visual controls for pausing and resuming agents and nests.

### Pause/Resume an Individual Agent

1. Navigate to the **Agent Detail** page for the agent you want to pause
2. Look for the **Pause/Resume button** in the header (next to the agent name)
   - The button shows a ⏸ (pause) icon when the agent is running
   - The button shows a ▶ (play) icon when the agent is paused
3. Click the button to toggle the agent's pause state

**Visual indicators on the Agent Detail page:**
- A **Paused badge** (⏸ Paused) appears next to the agent name when paused
- The badge has an amber background to indicate the paused state
- Hovering over the badge shows: "This agent runs no LLM turns until resumed"

### Pause/Resume an Entire Nest

1. Navigate to the **Nest Detail** page
2. In the header, you'll see two buttons:
   - **Nest pausieren** (Pause Nest) — pauses all agents in the nest
   - **Fortsetzen** (Resume) — resumes the nest
3. Click **Nest pausieren** to pause all agents at once
   - A confirmation toast appears: "Nest pausiert — alle Agents idle"
4. Click **Fortsetzen** to resume all agents
   - A confirmation toast appears: "Nest fortgesetzt"

**Notes:**
- The nest-wide pause is a "kill-switch" — it pauses every agent on the nest
- Individual agent pause states are preserved when the nest is resumed
- Buttons are disabled if the nest has no agents

### Troop Database Column

The troop maintains an `agents.paused` column that mirrors the nest registry. This column:
- Is updated by the same pause/resume endpoints used by the UI and CLI
- Ensures the paused badge displays correctly on page load
- Is populated via an idempotent migration (ALTER TABLE)
